GROUP PROJECT

USER TESTING METHODS RESEARCH

DELIVERABLE CONTAINING THE PROCESS OF USER RESEARCH METHODS RESEARCH FOR THE USER TESTING PLAN.

INTRO

This deliverable page presents a summary of the research on user testing methods which has been done by me and another group member. It is suggested to read through this summary first for a better understanding, after which to use the button at the bottom of the page to refer to as the actual proof document. The linked research question to this deliverable is:

  • How can we test the UI/UX of the app?

CONTEXT

When starting out with the user testing plan, there was still no design, and I had no idea what was going to be tested. Still, I thought it was a good idea to go on the hunt for possible research methods which might be suitable for this project.

EXECUTION

To start, a distinction was made between several types of research methods (Rohrer, 2024):

  • Attitudinal or behavioral
  • Moderated or unmoderated
  • Remote or face-to-face

After this, several research methods were chosen for which these types are applicable:

  • Usability testing
  • Guerrilla testing
  • A/B testing
  • Card Sorting
  • 5-second test
  • Eye Tracking
  • Screen Recording
  • Contextual Inquiry
  • The System Usability Scale (SUS)

By having a clear overview on what each method does, and its advantages and disadvantages, a decision on which method(s) would be used during the actual conducting of user testing would be easier to make. As for now, the list of research methods could be slimmed down to four specific research methods:

  • Usability testing
  • A/B testing
  • 5-second test
  • The System Usability Scale (SUS)

CONCLUSION & NEXT STEPS

After having a clear overview on potential user testing methods, a list of action points for the rest of the project rolled out. After sitting with Rose, some aspects that in her eyes needed to be looked at were added to this list of action points. These action points were as follows:

  • Working on the prototype for the app, and based on what needs to be tested, choose research method(s)
  • Cite used sources in future research documents.

REFERENCES

Rohrer, C. (2024b, januari 12). When to use Which User-Experience research methods. Nielsen Norman Group. https://www.nngroup.com/articles/which-ux-research-methods/
