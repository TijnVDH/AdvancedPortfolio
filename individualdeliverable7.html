PORTFOLIO DEVELOPMENT

DELIVERABLE CONTAINING THE DEVELOPMENT PROCESS OF THE PORTOLIO.

INTRO

For the development of my portfolio, I used vanilla HTML, CSS and JavaScript. After my front-end framework research, I had concluded that this would be the best way to work on the development of this portfolio.

START: HOMEPAGE

For the homepage, I wanted to achieve a look that invites the user to find out more about the portfolio. This would entail the logo and proof button at the top of the screen, and inviting content in the middle of the screen.

I re-used a lot of code from my already existing portfolio from my internship. While a lot of functionalities act the same, I did try to apply a different type of style to this portfolio, while still keeping the dark theme.

COMPETENCE TABLE

Working on the competence table was easier than I had anticipated by using the HTML table tags. After experimenting with different rows and columns I got to the final result I was hoping for:

PROOF SECTION

There is a proof section at the bottom of each project page. I had to make sure that The meaning of the deliverable, the attached learning outcome and an illustration for this was clear. By making use of a YouTube tutorial, I quickly got to the desired end-product.

VERSION CONTROL

I used Visual Studio's version control to keep my portfolio stored. If something went wrong during development, I could easily revert changes through Github.
